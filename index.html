Platform pillars

Built like a production exchange so our capstone feels ready to launch.

Registry-grade verification

Our verifier wallet scrapes county APIs + US Census geocoding, leaving on-chain evidence for every listing. No self-reported titles.

Dynamic marketplace

English auctions with reserve, min increment, buy-now, and anti-sniping extensions keep bids fair while pending returns protect bidders.

Autonomous escrow

Keeper bot calls `completeEscrow` the moment Title Registry shows the buyer as owner. If sellers miss deadlines, refunds trigger automatically.

Agent economics

Commission contract accrues earnings per listing and routes payouts with one tap. Agents can bid for themselves or clients on the same UI.

Workflow, end-to-end

Every team member follows the same timeline, so there’s no confusion about what happens next.

1

Submit property package

Seller uploads metadata + optional assessor links. Transactify emits a verification request to the oracle.

2

Oracle verdict

Verifier wallet cross-checks public data, signs an approval/rejection, and publishes evidence URIs on-chain.

3

List + bid

Sellers choose reserves/min increments or a buy-now. Bidders deposit ETH; outbid users withdraw instantly.

4

Escrow automation

After the timer, the escrow bot monitors title, releases funds to seller/agent, or refunds the buyer if deadlines pass.

Automation suite

The verifier oracle and escrow keeper run alongside the dApp so you can trust the flow even when everyone logs off.

Property Verifier

Node script that listens for events, queries US Census + municipal datasets in real time, then writes `recordVerificationResult` from a trusted wallet.

Escrow Keeper

Automation bot tracks each escrow window, calling `completeEscrow` or `claimEscrowRefund` so funds never linger.

Observability

Both services log to the console and can be wired into Slack/Webhooks for production observability.
